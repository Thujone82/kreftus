TemPy v1.5

TemPy is a cross-platform Python terminal UI for ThermoPro TP357/TP358/TP359 Bluetooth hygrometer/thermometer units. It polls sensors over BLE on a 5-minute schedule, shows a live monitoring dashboard with 24-hour temperature and humidity sparklines, and logs readings to CSV. Built with Textual and bleak for Windows and Linux.

Features:

  • 24H BLE history: Pull minute-level history from the sensor to backfill sparklines — H on Manage Devices, optional Y/N when adding a device, automatic pull on startup when logging is off, or --history-day on the CLI. Merges only the received time span so older polled/log data is preserved after a sensor reboot.
  • Monitoring dashboard: Five rows per device — name, temp stats, temp sparkline, humidity stats, humidity sparkline — with indoor comfort color bands on values and glyphs. Fetch arrows show connect (cyan), fast read (green), or legacy fallback (yellow).
  • Device management: Discover nearby TP35x units, add/rename/remove/reorder sensors, and open per-device status with 4H/24H/72H history sparklines.
  • Scheduled polling: Fetches on clock boundaries (:00, :05, …) with per-device retries and manual G fetch; sequential reads with a 60 s timeout per sensor.
  • Bluetooth recovery: If the OS reports Bluetooth is off (e.g. after sleep), TemPy power-cycles the radio and retries automatically.
  • Multi-column layout: Wide terminals can show multiple devices side by side — press C to cycle columns.
  • Quick device info: Keys 1–9 and 0 open the info panel for the first ten visible devices from the monitoring screen.
  • CLI modes: -x prints a log snapshot and exits; -nopoll / -np disables BLE scheduling (log reload every 5 min); -f / -filter limits the dashboard view by device name.

Place tp.ini and tp.log beside the launcher (tp.exe, tp.pyz, or tp.py). On first run with no devices configured, TemPy opens Manage Devices and starts discovery automatically.
